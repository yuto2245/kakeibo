Routerの役割と動作
# 1. Routerはパス管理を一括で行える
はい。Express Routerは、関連するエンドポイントをグループ化して管理できます。
具体的な動作の流れ
```
index.ts
  └─ app.use('/api/zaim', authRoutes)  ← ベースパスを設定
       │
       └─ authRoutes.ts
            ├─ router.post('/auth', requestAuth)     → /api/zaim/auth
            └─ router.get('/callback', handleCallback) → /api/zaim/callback
```
# 2. Routerが自動的にエンドポイントを定義する仕組み
router.post()やrouter.get()を使うと、その定義が自動的にエンドポイントとして登録されます。
詳しい説明
Routerの動作例

src/routes/authRoutes.ts（ルート定義ファイル）
```tsx
import { Router } from 'express';
import { requestAuth, handleCallback } from '../controllers/authController';

const router = Router();  // Routerインスタンスを作成

// Routerにエンドポイントを登録
router.post('/auth', requestAuth);           // POST /api/zaim/auth
router.get('/callback', handleCall);
```

src/index.ts（メインファイルでRouterを使用）
```tsx
import express from 'express';
import authRoutes from './routes/authRoutes';

const app = express();

app.use(express.json());

// Routerをベースパス '/api/zaim' でマウント
app.use('/api/zaim', authRoutes);

// これで以下のエンドポイントが自動的に定義されます：
// POST http://localhost:3000/api/zaim/auth
// GET  http://localhost:3000/api/zaim/callback
```
